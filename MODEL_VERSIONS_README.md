# Версии моделей (Model Versions)

## Описание

Этот экран позволяет сравнить три LLM-модели разной мощности (слабая, средняя, сильная) по одному и тому же запросу.

## Функциональность

### Основные возможности

- Параллельное выполнение запросов к трем моделям
- Сравнение по следующим метрикам:
  - Время ответа (latency)
  - Количество токенов (prompt, completion, total)
  - Стоимость запроса
  - Качество ответа (визуальная оценка)

### Модели

Конфигурация моделей находится в `app/src/main/java/com/example/aiadventchallenge/data/config/ModelVersionsConfig.kt`:

| Сила | Модель | ID | Цена ввода (1M) | Цена вывода (1M) |
|------|--------|-----|----------------|------------------|
| Слабая | DeepSeek R1 | `deepseek/deepseek-r1` | $0.55 | $2.19 |
| Средняя | MiniMax M2.7 | `minimax/minimax-m2.7` | $0.30 | $0.60 |
| Сильная | DeepSeek V3.2 | `deepseek/deepseek-v3.2` | $0.27 | $1.10 |

### Архитектура

- **AiRepository**: Единственная точка для всех HTTP запросов к AI API. Добавлен метод `askWithUsage()` для получения данных о токенах
- **AskModelUseCase**: Использует AiRepository для выполнения запросов к моделям, измеряет latency и рассчитывает стоимость. Добавлено детальное логирование
- **ModelVersionsConfig**: Конфигурация моделей с их ID и ценами
- **ModelResultsExporter**: Отдельный класс для экспорта результатов в файлы (JSON и Markdown). Принимает `File` (outputDir) через конструктор
- **ConclusionsGenerator**: Генерирует структурированные выводы с анализом качества, скорости и ресурсоёмкости

### Выходные файлы

Результаты сравнения автоматически сохраняются в директорию `/Download/` устройства:

- **results.json** - Полные результаты в формате JSON
- **summary.md** - Краткое сравнение в Markdown формате

**Путь к файлам:** `/storage/emulated/0/Download/` (или `/sdcard/Download/`)

Для просмотра файлов:
1. Откройте приложение "Файлы" или файловый менеджер
2. Перейдите в папку "Загрузки" (Downloads)
3. Найдите файлы `results.json` и `summary.md`

Или через ADB:
```bash
adb shell ls /sdcard/Download/
adb pull /sdcard/Download/results.json
```

Для мониторинга процесса экспорта используйте Logcat:
```bash
adb logcat | grep ModelResultsExporter
```

## Использование

### Запуск приложения

1. Убедитесь, что `AI_API_KEY` задан в `local.properties`:
   ```properties
   AI_API_KEY=your_api_key_here
   ```

2. Запустите приложение на устройстве или эмуляторе

3. Перейдите на вкладку "Модели" в нижнем меню

### Выполнение сравнения

1. Введите запрос в текстовое поле
2. Нажмите кнопку "Отправить"
3. Дождитесь выполнения запросов (индикатор загрузки)
4. Просмотрите результаты в карточках для каждой модели
5. Нажмите кнопку "Выводы" для просмотра детального анализа
6. Результаты автоматически сохранятся в папку `/Download/` устройства

### Обработка ошибок

- Если API-ключ неверный или отсутствует, появится сообщение об ошибке
- Если какая-то модель вернула ошибку, она будет отображена в карточке результата
- Если все модели вернули ошибки, появится соответствующее сообщение

### Логирование

Все запросы к моделям логируются с детальной информацией. Для просмотра логов:


## Кастомизация

### Изменение моделей

Отредактируйте `ModelVersionsConfig.kt`:

```kotlin
object ModelVersionsConfig {
    val models: Map<ModelStrength, ModelVersion> = mapOf(
        ModelStrength.WEAK to ModelVersion(
            strength = ModelStrength.WEAK,
            modelId = "your/model-id",
            modelName = "Ваша модель",
            inputPricePer1M = 0.50,
            outputPricePer1M = 1.00
        ),
        // ...
    )
}
```

### Изменение цен

Обновите значения `inputPricePer1M` и `outputPricePer1M` в конфигурации модели.

### Изменение API endpoint

Отредактируйте `ApiConfig.kt`:

```kotlin
data class ApiConfig(
    val baseUrl: String = "https://your-api-endpoint.com/api/v1/chat/completions",
    // ...
)
```

## API

### OpenRouter-compatible API

Используется стандартный формат запросов OpenRouter. Для работы требуется API ключ от провайдера, поддерживающего этот формат.

### Заголовки запроса

- `Authorization: Bearer {API_KEY}`
- `Content-Type: application/json`
- `HTTP-Referer: https://example.com`
- `X-Title: AiAdventChallenge`

## Зависимости

- Kotlin Coroutines для асинхронных запросов
- Kotlin Serialization для JSON
- OkHttp для HTTP запросов
- Jetpack Compose для UI
- Jetpack ViewModel для управления состоянием

## Особенности реализации

- **Параллельное выполнение**: Три запроса выполняются одновременно с использованием `coroutineScope` и `async`
- **Измерение времени**: Latency измеряется от начала запроса до получения ответа
- **Расчет стоимости**: Стоимость вычисляется на основе токенов и цен за 1M токенов
- **Экспорт результатов**: Автоматическое сохранение в JSON и Markdown
- **Обработка ошибок**: Graceful handling API errors с сохранением в results
- **Архитектура**: AskModelUseCase использует AiRepository для выполнения запросов, а не прямые HTTP вызовы
- **Единый репозиторий**: AiRepository содержит всю логику работы с API, включая новый метод `askWithUsage()`
- **Разделение ответственности**: Экспорт результатов вынесен в отдельный класс `ModelResultsExporter` (data слой)
