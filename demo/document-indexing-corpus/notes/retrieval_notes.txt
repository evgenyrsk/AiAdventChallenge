Local retrieval notes:

The pipeline should load documents from a directory, normalize them into a
common parsed representation and then apply chunking independently from the
embedding provider. This separation keeps the system maintainable.

Each chunk should preserve enough information to be useful on its own. Chunks
must also remain traceable to the original document, section and source.

Retrieval quality is easier to improve later when storage is not tightly coupled
to a specific embedding backend or a single chunking strategy.

SQLite is a practical default local store because it is easy to ship, inspect
and migrate. JSON is useful for debugging and exporting. FAISS can remain an
extension point for vector-heavy scenarios.

The first retrieval iteration should optimize for debuggability. It is better to
return transparent results with source metadata than to pursue premature ranking
complexity that is difficult to validate.
