Заметки по локальному retrieval:

Пайплайн должен загружать документы из директории, нормализовать их в общее
внутреннее представление и только потом применять chunking независимо от
embedding provider. Такое разделение делает систему более поддерживаемой.

Каждый чанк должен сохранять достаточно информации, чтобы быть полезным сам по
себе. При этом чанки должны оставаться привязанными к исходному документу,
секции и источнику.

Качество retrieval проще улучшать позже, когда хранилище не жёстко связано с
конкретным embedding backend или одной-единственной стратегией chunking.

SQLite — практичный вариант локального хранилища по умолчанию, потому что его
легко поставлять, проверять и мигрировать. JSON полезен для отладки и экспорта.
FAISS может оставаться точкой расширения для сценариев с тяжёлой векторной
нагрузкой.

Первая итерация retrieval должна оптимизироваться под отладку. Лучше возвращать
прозрачные результаты с metadata источника, чем слишком рано усложнять
ранжирование так, что его станет трудно проверять.
