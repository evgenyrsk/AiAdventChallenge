# Fitness Long Dialog Scenarios

Два длинных сценария для проверки production-like mini-chat с `RAG_ENHANCED` и `task memory`.

## Как прогонять

1. Открыть Android чат.
2. Выбрать режим `RAG Enhanced`.
3. На каждом ходе открыть debug details.
4. Зафиксировать:
   - `Task Memory`
   - `Original / Rewritten / Effective query`
   - `Grounded Sources`
   - `Grounded Quotes`
   - итоговый ответ

## Scenario 1: Fat loss with iterative constraints

Ожидаемая цель диалога:
- устойчивое снижение веса без потери мышц

Ожидаемые ограничения:
- времени мало
- пользователь уточняет белок, шаги, кардио, дефицит
- в конце нужен компактный итоговый план

Сообщения:
1. `Хочу снизить вес и не терять мышцы. С чего начать?`
2. `У меня мало времени, тренироваться могу по 30 минут.`
3. `А что насчет белка при похудении?`
4. `Сколько шагов стоит держать каждый день?`
5. `Можно ли худеть без кардио?`
6. `Какой дефицит калорий обычно считают разумным?`
7. `Если режим сорвется на несколько дней, что важнее вернуть первым?`
8. `Собери это в короткий практический план.`
9. `Сделай акцент на самом минимальном варианте.`
10. `И еще раз коротко: что по белку и шагам?`

Шаги, где `task state` должен обновляться заметно:
- 1: фиксируется goal
- 2: добавляется ограничение по времени
- 3-6: накапливаются clarifications
- 8-10: summary должен сжаться без потери goal/constraints

Проверки:
- follow-up `про белок` и `про шаги` не теряют goal снижения веса
- retrieval query учитывает цель и ограничения
- grounded answer на каждом шаге содержит `sources`

## Scenario 2: Beginner strength with sustainable routine

Ожидаемая цель диалога:
- начать силовые тренировки как новичок и построить устойчивый режим

Ожидаемые ограничения:
- новичок
- времени мало
- важны сон и восстановление
- цель не “масса любой ценой”

Сообщения:
1. `Хочу начать силовые тренировки, я новичок.`
2. `Сколько тренировок в неделю обычно достаточно новичку?`
3. `У меня мало времени, реально 3 короткие тренировки.`
4. `Как сон и восстановление влияют на такой режим?`
5. `Мне не нужна масса любой ценой, хочу устойчивый режим.`
6. `Какой сплит выбрать в таком случае?`
7. `Что важнее на старте: идеальный план или регулярность?`
8. `Собери мне краткий план на неделю.`
9. `Сократи его до самой простой версии.`
10. `Вернись к вопросу про сон: что критично не забыть?`

Шаги, где `task state` должен обновляться заметно:
- 1: фиксируется goal + constraint `новичок`
- 3: добавляется time constraint
- 4-5: добавляются recovery/sustainability constraints
- 8-10: summary и retained goal должны оставаться стабильными

Проверки:
- follow-up про сон не ломает фокус на beginner strength routine
- ответы не перескакивают в “массу любой ценой”
- grounded answer продолжает показывать `sources` и не смешивает их с task memory

## Report Template

Для каждого шага сценария фиксировать таблицу или JSON со следующими полями:

- `step`
- `userMessage`
- `taskStateBefore`
- `taskStateAfter`
- `retrievalQuery`
- `rewrittenQuery`
- `retrievedSources`
- `finalAnswer`
- `hasSources`
- `goalPreserved`
- `constraintsPreserved`
- `notes`
